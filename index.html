Caso de estudio

Portfolio web personal — landing responsive

Rol: diseño gráfico e identidad visual, maquetación web y criterios UX/UI en formación. Proyecto académico alineado al módulo de diseño visual / desarrollo web y a la entrega de empleabilidad.

1. Descripción de la actividad

Se planteó el diseño y la implementación de una landing page como portfolio personal: presentación, habilidades, proyectos destacados y contacto. El entregable debía ser responsive, con HTML semántico, estilos en CSS, uso de Bootstrap como apoyo a la rejilla y componentes, y documentación de decisiones de color, tipografía y estructura. El sitio debía reflejar un perfil de diseño gráfico con foco en identidad visual e incursión en UX/UI.

2. Desafío principal

Traducir una identidad coherente (paleta, tipografías, tono) a una interfaz web usable en móvil, tablet y escritorio, sin perder jerarquía ni legibilidad. El reto fue alinear prototipo y resultado final, mantener un código ordenado para quien recién integra desarrollo front-end, y comunicar con honestidad el nivel en UX/UI (en formación) frente al fuerte peso en diseño gráfico e identidad.

3. Solución propuesta

  1. Referencias y análisis de portfolios; definición de secciones (inicio, sobre mí, proyectos, contacto).
  2. Wireframes en media fidelidad (móvil y escritorio) y mockup de alta fidelidad para validar layout y estilo.
  3. Elección de paleta con buen contraste y par tipográfico (titulares / cuerpo) vía Google Fonts.
  4. Maquetación con HTML5 semántico, hoja CSS propia para identidad y Bootstrap 5 para grid, navbar, cards y formulario.
  5. Pruebas en distintos anchos de ventana; ajustes de espaciado, menú colapsable y textos alternativos en imágenes.

5. Principales aprendizajes

  • Cómo las variables CSS y un tema coherente con Bootstrap reducen inconsistencias entre navbar, botones y enlaces.
  • El valor de prototipar antes de codificar para anticipar problemas de jerarquía y scroll en móvil.
  • La importancia de la documentación escrita (justificación de color y tipografía) para el discurso profesional, no solo la pantalla final.
  • Límites actuales: formulario de contacto como demostración sin backend; próximo paso integrar un servicio de envío (Formspree, etc.) o enlaces directos.
